My Interest: Cybersecurity

Why I think every developer should understand security.

What is Cybersecurity?

Cybersecurity is the practice of protecting computers, networks, and data from unauthorised access or attack. In today's world, almost every application is connected to the internet, which means security is something every developer needs to understand.

Ethical hacking (also called penetration testing) is when security professionals legally break into systems with permission, to find weaknesses before real attackers do. It is one of the most exciting areas in tech.

I became interested in cybersecurity because I enjoy solving puzzles and thinking logically. Platforms like TryHackMe let me practise hacking skills in a safe, legal environment.

Common Cyber Threats

Here are some of the most important threats that developers should be aware of:

Phishing

Fake emails or messages that trick users into giving away passwords or clicking bad links.

SQL Injection

Malicious code inserted into a text field that can manipulate or destroy a database.

Ransomware

Malware that locks your files and demands payment to unlock them.

Cross-Site Scripting

Injecting scripts into a website that run in other users' browsers to steal data.

Why It Matters for Developers

Security should not be added at the end of a project - it should be part of every decision made during development. Here are the top practices every developer should follow:

  1. Validate all user input - never trust data sent by the user.
  2. Use parameterised queries - prevents SQL injection attacks.
  3. Use HTTPS - encrypts data sent between the browser and server.
  4. Keep software updated - outdated libraries are a common attack target.
  5. Least privilege - only give users the access they actually need.

My Goals in Cybersecurity

I am currently building my knowledge through online challenges and plan to sit the CompTIA Security+ certification in my second year. My long-term goal is to work as a junior penetration tester after graduating.
